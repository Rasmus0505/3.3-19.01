Zeabur Agent 不能做的事 ❌
代码相关
不能为你写代码
我是 DevOps 工程师，不是编程 AI
不能编写应用逻辑、业务代码、前端代码等
建议用编程 AI（如 Cursor、Claude Code）来写代码
不能修改你的源代码
不能编辑 package.json、requirements.txt 等配置文件
不能修改应用代码
只能读取和分析代码用于部署
不能自动优化代码
不能重构你的代码
不能修复代码中的 bug
不能性能优化你的应用
部署相关
不能使用非 Dockerfile 的构建方法
只能用 Dockerfile 部署
不能用 nixpacks、buildpacks 等其他构建系统
不能自动检测和使用其他构建工具
不能部署到非 Zeabur 平台
只能部署到 Zeabur
不能部署到 AWS、GCP、Heroku 等其他平台
不能管理多个平台的账户
只能操作你的 Zeabur 账户
不能访问其他云平台的账户
基础设施相关
不能购买或管理服务器
不能为你购买 VPS
不能管理服务器的物理配置
只能在你已有的服务器上部署服务
需要你手动去 https://zeabur.com/new 购买服务器
不能修改 Zeabur 平台本身
不能改变平台的功能
不能访问平台的后端系统
只能使用公开的 API
不能管理账户和支付
不能修改账户信息
不能处理支付和充值（除了查看余额）
不能管理订阅计划
数据库相关
不能直接操作数据库数据
不能执行 SQL 查询
不能修改、删除数据库中的数据
不能备份或恢复数据库
只能配置数据库连接参数
不能创建数据库结构
不能创建表、索引等
需要你的应用或迁移脚本来完成
只能部署数据库服务本身
安全相关
不能进行安全审计
不能深度分析代码安全问题
可以指导你使用 Zeabur 的安全扫描功能
不能修复安全漏洞（需要编程 AI）
不能管理 SSL/TLS 证书
Zeabur 自动管理 HTTPS 证书
不能手动配置证书
不能管理自定义域名的 DNS 记录
其他限制
不能执行写入命令
只能在容器内执行只读命令（ls、cat、grep 等）
不能安装包、修改文件、重启进程等
这是安全限制
不能提供技术支持
遇到 Zeabur 平台问题需要联系官方
可以指导你去 https://zeabur.com/forum 提问
不能解决平台 bug
