# 第01章：互联网与网站最小认知

## 1.1 先建立一张“全景图”

当你在浏览器输入一个网址并按下回车，背后最小流程是：

1. 浏览器解析 URL
2. DNS 把域名解析为 IP
3. 浏览器通过协议（HTTP/HTTPS）向服务器发请求
4. 服务器返回页面或 JSON
5. 浏览器渲染页面，或前端继续调用接口

---

## 1.2 术语精讲（入门必会）

### 互联网（Internet）

- 一个由大量网络互联组成的全球网络系统
- 重点不是“某台服务器”，而是“互联规则”

### Web

- 互联网里用于访问网页和 API 的一套体系
- 主要依赖 HTTP/HTTPS

### 网站（Website）

- 对用户可访问的一组页面和服务
- 通常由“前端 + 后端 + 数据库”组成

### 浏览器（Browser）

- 你与网站交互的客户端程序（Chrome、Edge）
- 能发 HTTP 请求、执行前端 JavaScript、渲染页面

### 服务器（Server）

- 接收请求并返回响应的程序或机器
- 在本项目里，后端服务由 FastAPI 提供

### 客户端（Client）

- 发起请求的一方
- 浏览器是客户端，手机 App 也是客户端

### 域名（Domain）

- 给 IP 地址起的人类可读名字
- 例：`example.com`

### IP 地址

- 网络中的机器地址
- IPv4 例：`203.0.113.8`

### DNS

- 负责把域名翻译成 IP 的系统
- 没有 DNS，你就得记 IP 才能访问服务

### 端口（Port）

- 同一台机器上区分不同服务的“门牌号”
- 例：`8000` 常见于开发环境 Web 服务

### 协议（Protocol）

- 通信双方遵守的规则
- Web 最关键的是 HTTP/HTTPS

### HTTPS

- HTTP + TLS 加密
- 防止中间人窃听内容

## 1.3 “前端 / 后端 / 数据库”三层关系

可以把系统理解成餐厅：

- 前端：点餐界面（用户操作入口）
- 后端：后厨与服务员（处理业务逻辑）
- 数据库：仓库与账本（保存数据）

三者关系图：

```text
用户
  │
浏览器(前端 React)
  │ HTTP 请求
后端(FastAPI)
  │ SQL
数据库(Postgres)
```

## 1.5 常见误区

### 误区 1：有网页就等于只有前端

错误。前端只是 UI 层，没有后端就无法做登录、上传、转写、持久化。

### 误区 2：端口就是域名

错误。域名用于定位主机，端口用于定位主机上的具体服务。

### 误区 3：部署等于买服务器

对本项目不成立。这里是 GitHub + Zeabur 托管部署，用户不需要自建运维体系。
