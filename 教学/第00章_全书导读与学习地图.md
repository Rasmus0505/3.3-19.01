# 第00章：全书导读与学习地图

## 0.1 这本书解决什么问题

这套文档的目标不是“只教你把项目跑起来”，而是让你从**入门概念**一路走到**能独立改项目、独立上线、独立排障**。

你会同时得到两条能力线：

1. 计算机与 Web 基础能力（术语、原理、常见坑）
2. 本项目实战能力（看懂代码、改功能、在 Zeabur 部署）

---

## 0.2 读者画像与学习方式

你是这类读者：

- 会 GitHub，能提交代码
- 会使用 Zeabur，但不熟悉服务器运维
- 想把“概念”真正转成“能交付”

建议学习节奏：

- 每章先读“术语解释”
- 再读“本项目对应位置”
- 最后完成“章节自测”

如果你是纯新手，建议按章节顺序阅读；如果你在排障，可先跳到第 8、9、10 章。

## 0.4 本项目一句话认知

这是一个英语句级拼写训练 MVP：

1. 上传音视频
2. 调用 DashScope 做 ASR
3. 自动切句与翻译
4. 在前端进入沉浸式逐句练习
5. 管理员可管理计费与兑换码

核心接口中最关键的是：

- `POST /api/transcribe/file`（上传并生成课程）
- `GET /health`（进程活着）
- `GET /health/ready`（数据库与业务就绪）

## 0.6 你会反复看到的关键概念

- `前端`：用户看得见、可操作的界面（React）
- `后端`：接收请求、处理业务、返回数据（FastAPI）
- `数据库`：持久化数据（Postgres）
- `容器`：把运行环境和代码打包在一起（Docker）
- `环境变量`：部署时注入配置（如 API Key）
- `接口契约`：前后端约定的路径与字段，不可随意破坏
