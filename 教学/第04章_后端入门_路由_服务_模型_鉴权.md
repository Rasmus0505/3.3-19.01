# 第04章：后端入门（路由、服务、模型、鉴权）

## 4.1 后端在本项目中的职责

后端负责：

1. 暴露 API 给前端调用
2. 处理业务逻辑（上传、转写、切句、计费）
3. 与数据库交互，持久化数据
4. 做鉴权与权限控制

本项目后端框架：FastAPI。

---

## 4.2 先看入口：`app/main.py`

入口文件做了几件关键事：

1. 创建 FastAPI 应用
2. 注册路由（`include_router`）
3. 挂载静态资源（前端构建产物）
4. 定义 `/health`、`/health/ready`
5. 在生命周期中做运行态探测

这意味着：一个服务同时承载 API 与前端静态页面。

---

## 4.3 路由（Router）是什么

路由就是“路径到处理函数的映射”。

示例认知：

- `POST /api/transcribe/file` -> 上传并生成课程
- `GET /api/lessons` -> 查询课程列表
- `DELETE /api/lessons/{lesson_id}` -> 删除课程

路由层职责应尽量薄：

- 参数校验
- 调服务层
- 返回标准响应

---

## 4.4 服务层（Service）是什么

服务层封装“业务流程”，比如：

- `LessonService.generate_from_upload(...)`
- `transcribe_uploaded_file(...)`

它们通常会串联多步骤：

1. 保存上传文件
2. 音频处理
3. 调 ASR
4. 切句与翻译
5. 写入数据库
6. 返回课程结果

---

## 4.5 模型（Model）与 Schema 的区别

### Model（`app/models/`）

- ORM 模型，对应数据库表结构
- 用于数据库读写

### Schema（`app/schemas/`）

- API 入参与出参的数据结构
- 用于请求/响应校验与序列化

简化理解：

- Model 面向数据库
- Schema 面向接口

---

## 4.6 仓储层（Repository）是什么

仓储层常见于 `app/repositories/`，作用是集中封装数据库查询逻辑：

- 哪些 SQLAlchemy 查询组合
- 如何过滤用户数据
- 如何分页/排序

优点：

- 服务层更聚焦业务
- 查询逻辑更容易复用与测试

---

## 4.7 鉴权与权限

本项目采用 token 鉴权思路：

1. 用户登录后拿到 token
2. 请求头携带 `Authorization: Bearer ...`
3. 后端依赖项解析当前用户
4. 普通用户只能操作自己的课程
5. 管理员接口需额外管理员身份检查

常见依赖在 `app/deps.py` 暴露。

---

## 4.8 错误处理思路

后端错误通常分三层：

1. 业务可预期错误（参数、权限、余额不足）
2. 上游依赖错误（ASR 服务异常）
3. 系统错误（数据库、文件系统异常）

应尽量返回可诊断信息，但避免暴露敏感细节。

---

## 4.9 健康检查为什么要分两级

- `/health`：进程存活，适合快速探活
- `/health/ready`：业务依赖可用，适合对外放量前判定

部署平台看到 `/health` 正常，不代表你就能正常上传转写。
