# 第06章：文件上传、ASR 转写、任务流

## 6.1 先理解“上传转写”到底做了几件事

用户看到的是“上传一个文件，生成一节课程”。  
系统实际做的是一条流水线：

1. 接收文件
2. 做媒体预处理（必要时转码）
3. 调用 ASR 识别文本与时间戳
4. 切句并生成 token
5. 翻译中文
6. 写入课程与句子
7. 返回课程详情给前端

---

## 6.2 核心接口契约

本项目核心接口必须保持稳定：

- `POST /api/transcribe/file`

契约稳定的含义：

- 路径不要随意改
- 核心请求字段和响应结构不要随意改名
- 前端、测试、部署脚本通常都依赖这个契约

---

## 6.3 后端入口在什么位置

路由入口：

- `app/api/routers/transcribe.py`

路由内部会调用：

- `app/services/transcription_service.py`
- `app/services/lesson_service.py`
- `app/services/lesson_builder.py`

你可以把它们理解为：

- `transcribe` 路由：总入口
- `transcription_service`：ASR 调用与音频处理
- `lesson_service`：课程生成编排
- `lesson_builder`：句子提取与切分细节

---

## 6.4 上传流程图（简化）

```text
前端 UploadPanel
  │ POST /api/transcribe/file
  ▼
transcribe router
  │ 参数校验 + 超时控制
  ▼
LessonService.generate_from_upload
  │
  ├─ 保存临时文件
  ├─ 音频处理(ffmpeg)
  ├─ 调 DashScope ASR
  ├─ 切句 + 翻译
  ├─ 写数据库
  ▼
返回 lesson + sentences
  │
前端刷新课程列表并进入学习
```

## 6.5 关键术语

### ASR（Automatic Speech Recognition）

- 自动语音识别
- 把音频里的语音转成文本

### 切句

- 把长段文本按时间片段拆分成学习句子
- 每句通常带 `begin_ms`、`end_ms`

### Token

- 句子拆分后的最小学习单元（常见是单词级）
- 用于拼写练习对比

### 超时控制

- 上传和转写可能耗时较长
- 需要设置超时，避免请求无限挂起

## 6.6 失败场景与处理思路

### 场景 1：模型参数不支持

- 现象：返回 `400 INVALID_MODEL`
- 处理：前端回退到支持列表中的模型

### 场景 2：ASR 上游失败

- 现象：返回 `502` 相关错误
- 处理：检查 `DASHSCOPE_API_KEY`、上游状态、请求内容

### 场景 3：请求超时

- 现象：返回 `504 REQUEST_TIMEOUT`
- 处理：优化输入文件大小、检查服务资源、调整策略

### 场景 4：媒体处理失败

- 现象：媒体错误映射
- 处理：检查 `ffmpeg` 是否可用，检查文件格式

## 6.7 与计费/钱包的关系（业务视角）

上传转写通常伴随点数扣减或费率计算。  
所以“上传接口”不是孤立功能，它和以下模块联动：

- 计费费率
- 钱包余额
- 流水记录

如果出现“上传失败却扣费”或“上传成功未扣费”，要重点排查事务边界与异常分支。

---

## 6.8 前端如何消费结果

上传成功后，前端通常会：

1. 把新课程加入列表
2. 刷新当前课程详情
3. 跳转或进入沉浸学习区
4. 同步钱包余额变化
