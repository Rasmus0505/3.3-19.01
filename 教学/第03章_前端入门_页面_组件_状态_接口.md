# 第03章：前端入门（页面、组件、状态、接口）

## 3.1 前端在本项目中的职责

前端负责：

1. 展示课程、句子、进度、余额等信息
2. 提供交互（上传、播放、输入拼写、兑换码）
3. 调后端 API，拿业务数据并更新界面

本项目前端技术栈：

- React
- Vite
- Tailwind（样式）

---

## 3.2 目录怎么读

关键目录：

- `frontend/src/main.jsx`：前端入口
- `frontend/src/app/`：应用壳与页面编排
- `frontend/src/features/`：按业务拆分功能模块

推荐阅读顺序：

1. `main.jsx`
2. `app/LearningShell.jsx`
3. 对应 feature 组件（如 `upload`、`immersive`、`auth`）

## 3.3 核心术语逐条解释

### 组件（Component）

- React 的最小 UI 单元
- 可以复用，可组合

例子：`UploadPanel`、`LessonList`、`ImmersiveLessonPage`

### Props

- 父组件传给子组件的参数
- 子组件通过 props 接收数据和回调

### State

- 组件内部状态
- 状态变化会触发重新渲染

### 渲染（Render）

- React 根据状态生成界面
- 状态不变，界面通常不变

### 事件处理

- 点击、输入、键盘事件触发逻辑
- 常见：`onClick`、`onChange`、`onKeyDown`

## 3.4 本项目里的页面协作

在学习页中，典型结构是：

- 左侧：课程列表
- 中间：沉浸学习区
- 右侧：登录/上传/兑换码

一个典型交互：

1. 上传成功
2. 前端收到后端返回的课程数据
3. 更新课程列表与当前课程
4. 进入沉浸页开始学习

---

## 3.5 API 调用在前端怎么发生

最常见模式：

1. 组装请求（URL、方法、token、参数）
2. 发请求
3. 根据状态码或 `ok` 字段处理成功/失败
4. 更新 state 并重绘 UI

常见失败处理：

- token 过期：提示重新登录
- 参数错误：弹出错误信息
- 后端异常：展示“稍后重试”

## 3.6 新手最容易踩的前端坑

### 坑 1：直接改对象，界面不更新

React 依赖状态引用变化。不要直接改旧对象，应该创建新对象再 set。

### 坑 2：异步结果覆盖新状态

两个请求返回顺序不一定一致。后发请求可能先返回，导致旧数据覆盖。

### 坑 3：把业务逻辑都堆在一个组件

会变成巨型组件，难改难测。应按 feature 逐步拆分。
