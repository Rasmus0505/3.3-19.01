# Requirements v2.4: 词汇等级预处理与 CEFR 沉浸式展示

**Archived:** 2026-04-04
**Milestone:** v2.4
**Status:** ✅ ALL COMPLETE

## Requirements

### CEFR Analysis

- [x] **CEFR-01**: User can preprocess all subtitle text when opening a lesson — each word is tagged with its CEFR level (A1/B1/B2/C1/C2/SUPER) via local vocabAnalyzer lookup → Phase 24 Plan 04
- [x] **CEFR-02**: Preprocessing results are cached in localStorage keyed by lessonId with version prefix (`cefr_analysis_v1:{lessonId}`) — subsequent opens skip re-analysis → Phase 24 Plan 04
- [x] **CEFR-03**: Batch analysis runs in chunked mode (`setTimeout(0)` or `requestIdleCallback`) to avoid blocking the UI thread during long videos → Phase 24 Plan 04
- [x] **CEFR-04**: Unknown words (not in cefr_vocab.json) default to SUPER level so they always appear as "hard" → Phase 24 Plan 04

### CEFR Display — Immersive Learning

- [x] **CEFR-05**: Immersive answer board displays both current sentence and previous sentence — each word has a CEFR level badge overlay → Phase 25 Plan 02
- [x] **CEFR-06**: CEFR badge colors are distinct from existing letter-state colors: teal/blue for i+1 (within reach), amber/orange for above i+1 (too hard) — NO overlap with green/red/yellow letter states → Phase 25 Plan 01
- [x] **CEFR-07**: CEFR badge is a word-level overlay (covers entire word) — does NOT override letter colors (green=correct, red=wrong, yellow=hint remain visible) → Phase 25 Plan 02
- [x] **CEFR-08**: CEFR color calculation: green badge when word level == user_i_level + 1; yellow badge when word level >= user_i_level + 2; no badge for words at or below user_i_level → Phase 25 Plan 02
- [x] **CEFR-09**: CEFR display contract (exact colors, badge shape, z-index layering) is defined in UI-SPEC.md before any rendering code → Phase 25 Plan 01 (UI-SPEC.md)

### CEFR Display — History & Badges

- [x] **CEFR-10**: Lesson history list shows a CEFR badge on each lesson card — color block + level text (e.g., B1, B2) — read from cached analysis results → Phase 25 Plan 04
- [x] **CEFR-11**: Lesson-level CEFR distribution is calculated as aggregate from preprocessed word levels (e.g., "B1: 45%, B2: 30%, C1: 15%, Other: 10%") → Phase 25 Plan 04

### Personal Center — i Level Setting

- [x] **CEFR-12**: Personal center exposes a CEFR level selector — user sets their i level (default B1, options: A1/A2/B1/B2/C1/C2) → Phase 24 Plan 03
- [x] **CEFR-13**: Level selector includes Duolingo-style Chinese descriptions for each level (e.g., "A1: 能理解和使用熟悉的日常表达和非常简单的句子") → Phase 24 Plan 03
- [x] **CEFR-14**: i level is stored in user profile (PATCH /api/users/me or equivalent) so it persists across devices → Phase 24 Plan 01
- [x] **CEFR-15**: i level is also cached locally (Zustand persist) for offline use — synced with server on next online → Phase 24 Plan 02

### Wordbook Selection Interaction

- [x] **CEFR-16**: Tapping/selecting a word from the previous sentence to add to wordbook triggers a smooth scale-up animation (1.0 → 1.08, 200ms ease-out) as the primary feedback signal → Phase 25 Plan 03
- [x] **CEFR-17**: After a word is added to wordbook, its feedback distinguishes "selected for wordbook" from "CEFR difficulty color" — use scale + border/badging, NOT background color change → Phase 25 Plan 03
- [x] **CEFR-18**: Hover state on selectable words in previous sentence shows subtle scale + cursor hint → Phase 25 Plan 03
