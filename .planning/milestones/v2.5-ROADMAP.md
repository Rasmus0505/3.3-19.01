# v2.5 Roadmap — 阅读板块 + Pretext CEFR 排版

## 设计原则

**本地优先 — 压力转移到用户本地，服务器零压力：**
- CEFR 分析：vocabAnalyzer 在浏览器端执行，结果存 localStorage
- Pretext 测量：结果存 localStorage，不上传原文到服务器
- AI 重写：服务器仅存储 rewrite_id 引用，原文和重写结果存 IndexedDB
- 服务器职责：最小必要数据（rewrite_id + lesson_id 映射）

## Phase Detail

### Phase 26: Pretext 基础设施集成

**目标**: 将 `@chenglou/pretext` 集成进 frontend，建立 CEFR-aware 的测量管道。**所有测量在浏览器 localStorage 执行，服务器零压力。**

**Plans:**
- 26-01: 安装 `@chenglou/pretext`，验证包可正常 import
- 26-02: 封装 `usePretext` 和 `useRichLayout` hooks，建立 CEFR 分段合并逻辑（vocabAnalyzer 在浏览器端执行）
- 26-03: 实现 prepare 结果 localStorage 缓存（Map 缓存 key: `text+font+width`）
- 26-04: 长文本性能测试（5000词），验证 layoutWithLines 响应时间 < 50ms

**关键交付物:**
- `frontend/src/hooks/usePretext.ts` — 基础测量 hook
- `frontend/src/hooks/useRichLayout.ts` — 丰富行布局 hook（含 CEFR 元数据）
- 性能测试报告（Chrome DevTools Timeline 截图）

**Plans:**
