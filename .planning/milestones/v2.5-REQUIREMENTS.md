# v2.5 Requirements — 阅读板块 + Pretext CEFR 排版

## Milestone

**v2.5** — 阅读板块 + Pretext CEFR 排版

## Overview

新增阅读板块，使用 Pretext (`@chenglou/pretext`) 作为文字测量和排版引擎，在文章中以 CEFR 颜色标注每个词，实现边读边选词加入生词本的核心交互链路。

## Decision Backlog

| 决策项 | 选项 | 建议 | 状态 |
|--------|------|------|------|
| 技术路径 | Pretext `layoutWithLines` + React 渲染 | 方案 1：Pretext 测量+行定位，React 渲染+样式 | ✅ 确认 |
| 页面结构 | 方案 A：右侧词边栏 | 桌面：固定右侧边栏(280px)；平板/移动：底部抽屉 | ✅ 确认 |
| 单词交互 | 可点击、可多选 | 单词可点击加入生词本，支持多选 | ✅ 确认 |
| 划词划句 | 为后续翻译做准备 | 数据结构层预留，但 v1 不实现划选 UI | ✅ 确认 |
| 文章字数 | 适配所有场景 | 短文/中篇/长篇（5000+词）均需支持 | ✅ 确认 |
| 对照模式 | 原文 vs 重写版 | ✅ 单区域+顶部切换开关，无分屏，丝滑切换 | ✅ 确认 |
| 长按划词 | 选句子/段落翻译 | 数据结构预留，v1 UI 不实现 | ❓ 待确认 |

## Requirements

### 阅读模块 REQ

- [ ] **REQ-READ-01**: 用户可以从历史/课程进入阅读板块，阅读板块以方案 A（右侧词边栏）布局呈现
- [ ] **REQ-READ-02**: 文章正文由 Pretext (`@chenglou/pretext`) 驱动排版，每个词为独立可点击元素
- [ ] **REQ-READ-03**: 每个词按 CEFR 等级着色（绿=i+1，红=超纲，灰=已掌握），与沉浸式学习视觉一致
- [ ] **REQ-READ-04**: 用户点击任意单词，该词进入右侧「本词待学」列表，支持多选
- [ ] **REQ-READ-05**: 右侧词栏提供「加入生词本」「重写全文」「开始学习」三个操作按钮；阅读区顶部有「原文 / 重写版」切换开关
- [ ] **REQ-READ-06**: 右侧词栏显示当前文章的难度等级、总词数、超纲词数
- [ ] **REQ-READ-07**: 响应式布局：桌面（≥768px）右侧固定边栏；平板/移动边栏折叠为底部抽屉
- [ ] **REQ-READ-08**: 支持长文章（5000+词）流畅滚动，行高精确，无内容跳动

### 技术基础设施 REQ

- [ ] **REQ-PRETEXT-01**: `frontend` 安装 `@chenglou/pretext` 依赖
- [ ] **REQ-PRETEXT-02**: 封装 `usePretext` hook，提供 `prepare(text, font)` 和 `layout(prepared, width, lineHeight)` 的 React 集成
- [ ] **REQ-PRETEXT-03**: 封装 `useRichLayout` hook，提供 `prepareWithSegments` + `layoutWithLines` 的 React 集成，返回每行的词列表
- [ ] **REQ-PRETEXT-04**: CEFR 分析结果（`vocabAnalyzer`）与 Pretext 分段结果合并，每个 segment 带 CEFR 元数据
- [ ] **REQ-PRETEXT-05**: Pretext 测量结果缓存：相同 text+font+width 的 prepare 结果在浏览器 localStorage 中缓存，服务器不存储原文或测量结果

### 生词本集成 REQ

- [ ] **REQ-WB-01**: 阅读板块选中的词通过已有 `wordbookApi.addWord` 接口加入生词本
- [ ] **REQ-WB-02**: 词加入生词本后触发 scale + border flash 动画（复用 Phase 25 动画）
- [ ] **REQ-WB-03**: 阅读板块底部的「开始学习」按钮跳转到生词本复习页面

### 页面路由 REQ

- [ ] **REQ-ROUTE-01**: `/reading/:lessonId` — 阅读板块页面（从历史课程进入）
- [ ] **REQ-ROUTE-02**: `/reading/new` — 新建阅读（粘贴/导入文章）
- [ ] **REQ-ROUTE-03**: 导航栏「学习」菜单新增「阅读」入口

### AI 重写 REQ

- [ ] **REQ-REWRITE-01**: 右侧「重写全文」按钮调用 AI 将原文改写为 i+1 难度的简化版本
- [ ] **REQ-REWRITE-02**: 重写结果存储在 IndexedDB（用户本地），服务器仅存储 rewrite_id 和 lesson_id 引用关系；原文和分析数据不离开用户设备
- [ ] **REQ-REWRITE-03**: 重写结果同样以 Pretext 排版并 CEFR 着色显示
- [ ] **REQ-REWRITE-04**: 阅读区上方有「原文 / 重写版」切换开关，单区域展示，点击丝滑切换，无页面跳转

## Out of Scope

- 划词/划句选区交互 — 用户明确不需要
- 对照模式分屏并排显示 — 改用单区域 + 顶部切换开关实现（已覆盖）

## Dependencies

- Phase 24/25 CEFR 基础设施（`vocabAnalyzer`，CEFR 颜色系统）
- 现有生词本 API（`wordbookApi`）
- 现有 AI 重写基础设施
- **本地优先原则**：所有 CEFR 分析和 Pretext 测量在浏览器 localStorage/IndexedDB 执行，服务器仅存储最小必要数据
