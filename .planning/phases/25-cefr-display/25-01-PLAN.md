<task type="auto">
  <name>Task 2: Create CefrBadge.jsx utility component</name>
  <files>frontend/src/features/immersive/CefrBadge.jsx</files>
  <action>
    Create `CefrBadge.jsx` in `frontend/src/features/immersive/` with the following exports:

    1. **`CEFR_LEVEL_ORDER`** constant: `["A1", "A2", "B1", "B2", "C1", "C2", "SUPER"]`

    2. **`getCefrLevelIndex(level)`** function: Returns 0-based index of a CEFR level in CEFR_LEVEL_ORDER (A1=0, A2=1, B1=2, B2=3, C1=4, C2=5, SUPER=6). Unknown levels default to index 6 (SUPER).

    3. **`computeCefrClassName(wordLevel, userLevel)`** function:
       - Get wordIndex = getCefrLevelIndex(wordLevel)
       - Get userIndex = getCefrLevelIndex(userLevel)
       - If wordIndex <= userIndex: return "cefr-mastered"
       - If wordIndex === userIndex + 1: return "cefr-i-plus-one"
       - If wordIndex >= userIndex + 2: return "cefr-above-i-plus-one"
       - Default: return "cefr-mastered"

    4. **`CefrUnderline`** React component (for current sentence letter cells):
       ```jsx
       export function CefrUnderline({ wordLevel, userLevel, children, className = "" }) {
         const cefrClass = computeCefrClassName(wordLevel, userLevel);
         return (
           <span className={`${cefrClass} ${className}`}>
             {children}
           </span>
         );
       }
       ```

    5. **`CefrWordBadge`** React component (for previous sentence word blocks):
       ```jsx
       export function CefrWordBadge({ wordLevel, userLevel, children, className = "" }) {
         const cefrClass = computeCefrClassName(wordLevel, userLevel);
         return (
           <span className={`immersive-wordbook-token ${cefrClass} ${className}`}>
             {children}
           </span>
         );
       }
       ```

    Import `readCefrLevel` from `../../app/authStorage` for reading default user level. Accept `userLevel` as optional prop — fallback to `readCefrLevel() || "B1"` if not provided.

    Note: For Phase 25 implementation, the components provide the class name computation. The actual rendering integration (wrapping tokens with these classes) happens in Plans 02 and 03.
  </action>
