# Plan 19-02: 上一句小喇叭播放状态与错误提示

## Goal

Fix Bug 2: When the user clicks the "previous sentence" speaker button and audio is unavailable, the app must not silently fail. It must show a clear error message. A three-tier fallback (clip → main video → TTS) provides the best user experience.

## Context

Phase 8 D-04 contract: "上一句硬中断" — clicking previous sentence must not crash. Phase 12 decision: speaker button only plays audio clip without navigating. Root cause: `requestPlayPreviousSentence` calls `playSentence` with `skipSeek:true`, which fails silently when `previousSentence.audio_url` is absent. No TTS fallback exists.

<action>
In `ImmersiveLessonPage.jsx`, create a TTS helper function `speakPreviousSentenceTTS` using Web Speech API:

```js
const speakPreviousSentenceTTS = (text, rate = 1.0) => {
  if (!window.speechSynthesis) return false;
  window.speechSynthesis.cancel(); // stop any ongoing speech (prevents StrictMode duplicate)
  const utter = new SpeechSynthesisUtterance(text);
  utter.lang = 'en-US';
  utter.rate = rate;
  window.speechSynthesis.speak(utter);
  return true; // synchronous — API available = request sent
};
```

Place this function in the component body (after hooks), using `selectedPlaybackRate` from closure. No external imports needed — `window.speechSynthesis` is browser-native.
</action>
<acceptance_criteria>

  - It returns `false` when `window.speechSynthesis` is undefined
  - It calls `speechSynthesis.cancel()` before `speak()` to prevent duplicates
  - It accepts `rate` parameter and sets `utter.rate`
</acceptance_criteria>

<action>
Modify `requestPlayPreviousSentence` to implement three-tier fallback:

1. **Tier 1 — Clip with main video fallback:** Call `playSentence(previousSentence, { initialRate: selectedPlaybackRate, rateSteps: [] }, { skipSeek: false })` — note `skipSeek: false` to allow fallback to main video
2. **Tier 2 — TTS:** If Tier 1 returns `{ok: false}`, call `speakPreviousSentenceTTS(previousSentence.text_en, selectedPlaybackRate)`. If TTS also fails, proceed to Tier 3.
3. **Tier 3 — Error:** If TTS returns `false`, call `setMediaError('上一句音频不可用，请稍后重试。')` and `dispatchSession({ type: SET_PHASE, phase: "typing" })`, then return without changing playback state.
4. **Success path:** On TTS success, dispatch `PLAYBACK_STARTED` with `phase: "playing"` and `translationDisplayMode: "previous"` so the UI shows the playing state.

Reference implementation from 19-RESEARCH.md lines 152-179.
</action>

  - If TTS unavailable, setMediaError is called with the Chinese string
  - On TTS success, PLAYBACK_STARTED is dispatched
  - No silent failure — every path is explicitly handled
</acceptance_criteria>
