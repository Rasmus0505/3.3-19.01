# Plan 19-03: 答题框颜色区分（AI 黄色 vs 用户绿色）

## Goal

Fix Bug 3: Immersive lesson answer box shows a yellow background when displaying AI/hint-generated content, and a green background when the user has typed something. Visual distinction helps users understand which content is "reference" vs "their input".

## Context

Phase 8 established the word snapshot system. The answer box renders from `wordInputs`/`currentWordInput` state. The color must be driven by React state (`answerBoxMode`) so it stays in sync across re-renders.

Drive the answer box container className directly from `answerBoxMode` state. Never use a ref for this — drive from state only.

Identify the keydown handler for the answer box (likely `handleKeyDown` or similar). In the handler, detect user typing:

```jsx
const handleKeyDown = (event) => {
  // Detect user typing (not modifier key, not special key)
  if (event.key.length === 1 && !event.ctrlKey && !event.metaKey && !event.altKey) {
    setAnswerBoxMode('user_typed');
  }
  // ... rest of existing handler
};
```

This catches any printable character input. System fills (via `revealWord`, `replayAssistance`, or `setWordInputs`) do NOT call this handler and will not trigger the switch, keeping the box amber when AI content is revealed.
</action>

  - No system-side fill (revealWord, setWordInputs, applyWordSnapshot) triggers `setAnswerBoxMode('ai_content')`
  - The mode persists until the user types again in a later sentence
</acceptance_criteria>
