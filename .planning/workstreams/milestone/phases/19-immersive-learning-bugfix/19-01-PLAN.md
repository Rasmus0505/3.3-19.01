# Plan 19-01: 答题框输入时切换倍速不触发重播

## Goal

Fix Bug 1: When the user is actively typing in the answer box (`sentenceTypingDone=false`), changing playback rate must not trigger auto-replay or clear the typed input.

## Context

Phase 8 D-10 contract: "切换倍速不得重置输入" (switching playback rate must not reset typed input). Root cause identified in 19-RESEARCH.md — the `autoAdvanceGuard` useEffect (Effect 3) fires unconditionally when `sentenceTypingDone=true`, even during an active typing session where a button click may have temporarily set `sentenceTypingDone=true`.
