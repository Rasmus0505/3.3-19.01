# Plan 19-04: 循环开关切换不触发重播

## Goal

Fix Bug 4: When the user toggles the loop (repeat) switch while actively typing a sentence, the typed input must not be cleared or interrupted. This is the same root cause as Bug 1 — the `autoAdvanceGuard` useEffect and any replay-triggering paths.

## Context

Bug 4 shares the same root cause as Bug 1: `requestReplayCurrentSentence` or the autoAdvanceGuard useEffect fires during an active typing session. Bug 1 fixed the `autoAdvanceGuard` guard and `SET_PLAYBACK_RATE` path. This plan covers the `SET_LOOP_ENABLED` path and any remaining replay-triggering buttons.
