## 网页转化分流

| Option | Description | Selected |
|--------|-------------|----------|
| 网页优先，桌面补充 | 主按钮尽量留在网页端，仅在少数硬边界才切去桌面端 | |
| 场景优先分流 | 根据素材/能力/阻塞原因动态决定主 CTA | ✓ |
| 桌面优先，网页兜底 | 默认更积极地把用户导向桌面端 | |

**User's choice:** 场景优先分流。
**Notes:** 用户不希望固定成网页优先或桌面优先，而是希望 CTA 根据当前用户场景动态切换。

### 桌面端主 CTA 触发条件

| Option | Description | Selected |
|--------|-------------|----------|
| 仅能力边界 | 只有 Bottle 1.0、链接导入、桌面专属能力时才主推桌面端 | |
| 能力边界 + 高风险素材 | 硬边界和高风险素材都主推桌面端 | ✓ |
| 只有 Bottle 1.0 | 除 Bottle 1.0 外尽量都让网页继续主跑 | |

**User's choice:** 能力边界 + 高风险素材。
**Notes:** 用户接受把超大文件、长时长、网络不稳定等高风险素材也纳入桌面端主 CTA 触发条件，而不是只作为轻提醒。

### 按钮主次规则

| Option | Description | Selected |
|--------|-------------|----------|
| 网页可执行时主按钮永远留在当前页 | “开始生成”永远是主按钮，桌面端只做补充入口 | |
| 按场景动态换主按钮 | 适合网页时主推网页，适合桌面时主推下载桌面端 | ✓ |
| 固定双按钮 | 长期同时并列展示“开始生成”和“下载桌面端” | |

**User's choice:** 按场景动态换主按钮。
**Notes:** 该选择与“场景优先分流”保持一致，要求主次 CTA 能随着边界/风险/阻塞原因切换，而不是长期固定。
