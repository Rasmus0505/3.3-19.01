# Phase 7 Copy Deck

> This file is the reusable wording pack for model cards, blocked states, admin/runtime naming, and migration away from old technical labels.

网页端不得执行 Bottle 1.0。网页端只能解释它、推荐它、引导下载桌面端，不能把它写成浏览器内的备选执行流程。

## 模型卡文案

### 主问题

- 标题：`选择学习素材质量`

### 模型卡主文案

- `Bottle 1.0：生成速度一般，字幕质量较好（客户端专属）`
- `Bottle 2.0：生成速度快，字幕质量好`

### 模型卡辅助解释

- Bottle 2.0：适合网页即用、快速起步、常规文件上传。
- Bottle 1.0：适合复杂素材、长时长内容、链接导入和更重的桌面端处理场景。

## 受阻场景文案

### 桌面专属能力

- `Bottle 1.0 网页不可直接执行，请下载桌面端继续。`
- `链接导入属于桌面端专属流程，请下载桌面端继续。`
- `当前素材体积较大 / 时长较长 / 网络风险较高，更推荐下载桌面端处理。`
- `当前是余额阻塞，不是能力边界；主恢复动作保持为充值后生成。`

### 按钮词

- `下载桌面端`
- `充值后生成`
- `继续网页生成`
- `查看 Bottle 1.0 说明`
- `稍后再试`

### 使用规则

- 当问题来自 Bottle 1.0、链接导入或高风险素材时，主按钮用 `下载桌面端`。
- 当问题来自余额不足时，主按钮只能用 `充值后生成`，不能改写成桌面端能力引导。
- 任何受阻文案都不能暗示“网页端切换一下就能执行 Bottle 1.0”。

## Admin / 诊断命名

- `Bottle 1.0 / Bottle 2.0 是主命名`
- `技术 model key 只能出现在副注释或诊断说明中`

### 推荐写法

- Bottle 1.0：本地运行时诊断
- Bottle 2.0：云端接口诊断
- 副注释里可补充 `desktop_local`、`cloud_api`、具体 model key，但不能盖过主命名

## 旧词到新词映射

- `选择字幕生成方式 -> 选择学习素材质量`
- `本机识别 -> Bottle 1.0（可作为解释性副文案）`
- `云端识别 -> Bottle 2.0（可作为解释性副文案）`
- `本地 Bottle 1.0 -> Bottle 1.0`
- `云端识别直传 -> Bottle 2.0 默认网页流程`
