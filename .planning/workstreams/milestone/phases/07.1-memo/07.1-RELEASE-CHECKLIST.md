# Phase 07.1 Release Checklist

## 公开视频成功导入

- [ ] 用 `YouTube` 公开视频验证桌面端可以完成导入、下载/媒体准备、课程生成和学习跳转
- [ ] 用 `B站` 公开视频验证相同链路
- [ ] 用 `常见播客页面` 验证公开链接导入
- [ ] 用 `公开视频直链` 验证公开直链导入

## 受限内容失败分流

- [ ] 使用一个登录态或受限内容链接，确认出现明确失败说明而不是绕过方案
- [ ] 失败态明确推荐 `SnapAny`
- [ ] 失败后仍保留当前表面可重试，不自动跳离流程

## 低技术感知控制面

- [ ] 普通导入流程里不出现 cookies / 下载源 / yt-dlp 更新入口
- [ ] 用户主表面只显示链接、标题、进度、取消、失败提示和继续生成动作
- [ ] `客户端诊断` 仍然存在，但停留在内部诊断入口而不是导入主流程

## 标题与学习闭环

- [ ] 公开链接解析到标题后，课程默认标题合理写入
- [ ] 没有把封面、作者、频道、来源平台做成新的主界面资产
- [ ] `成功后默认直达学习`
- [ ] lesson / progress 行为仍与普通素材生成保持同一 canonical flow

## 对外文案一致性

- [ ] 产品说明与界面都明确写出“支持常见公开视频链接”
- [ ] 产品说明与界面都没有承诺 cookies、账号登录、会员内容、受限内容导入
- [ ] 产品说明与界面都把 `SnapAny` 作为 secondary fallback，而不是主路径
