# Phase 07.1: Support Boundary Spec

**Phase:** 07.1-memo  
**Status:** Draft for execution  
**Last updated:** 2026-03-29

## 对外支持承诺

本阶段对外承诺固定为：

`支持常见公开视频链接`

这是一条桌面端正式学习工作流的承诺，不是对底层下载工具能力广度的承诺，也不等于“支持所有可被某个工具解析的网站”。

## 支持示例

用于设定用户预期的明确示例固定为：

- `YouTube`
- `B站`
- `常见播客页面`
- `公开视频直链`

这些示例用于帮助用户理解“公开链接”口径，不构成需要长期维护的全量平台兼容清单。

## 明确不支持

本阶段明确不支持以下能力：

- `不支持 cookies / 账号登录 / 会员内容 / 受限内容导入`
- `不支持播放列表或批量链接`
- `不支持把 Bottle 扩展成受限内容绕过工具`

如后续要进入这些能力，必须单独立项，而不是在 07.1 中顺手加入口。

## 失败分流

失败和边界命中时，产品必须使用以下固定消息或语义：

- `当前桌面工具暂不支持该链接，建议改用 SnapAny`
- `该链接可能需要登录或平台限制，建议改用 SnapAny`
- `未识别到可导入链接。`

同时固定以下分流规则：

- `SnapAny 是 secondary fallback`
- 失败后仍优先把用户留在当前产品表面，便于重试
- `失败后不引导用户进入 cookie、登录态或下载源设置`

## 内部诊断边界

面向内部或排障的能力继续收口在诊断层，不进入普通导入表面：

- `客户端诊断`
- `不向普通用户暴露 yt-dlp 更新入口`
- `不向普通用户暴露下载源选择`
- `不向普通用户暴露 cookie 管理`

这意味着普通用户在导入流程里只看到链接、标题、进度、取消、失败说明和后续生成动作，而不会看到媒体工具控制面。
