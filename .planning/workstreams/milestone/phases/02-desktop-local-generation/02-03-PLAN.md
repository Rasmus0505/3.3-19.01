# Objective

实现统一的错误处理：模型损坏时引导用户重新下载，不展示降级原因 Banner，helper 异常时显示统一失败状态。

## Context

根据 02-CONTEXT.md 决策：
- **D-01:** 用户完全不感知 ASR 来源，无技术细节暴露
- **D-02:** 无诊断面板、无 helper 状态显示、无模型状态指示器
- **D-05:** 错误恢复：显示统一失败状态。模型损坏时引导重新下载，不展示降级原因 Banner
- **D-12:** 模型损坏时：通过 model download/install flow 引导用户重新下载

关键决策（Claude's Discretion）：
- 统一失败文案：通用"生成失败，请重试"vs 具体"模型需要重新下载"
- helper 进程崩溃行为：自动重启 vs N次失败后显示失败

示例实现：
```javascript
export function isModelCorruptionError(error = {}) {
  const message = String(error?.message || error?.detail || "").toLowerCase();
  return message.includes("model") && 
         (message.includes("corrupt") || message.includes("invalid") || message.includes("missing"));
}

export function getModelRedownloadGuidance() {
  return "模型文件可能损坏，请重新下载后重试。";
}
```
