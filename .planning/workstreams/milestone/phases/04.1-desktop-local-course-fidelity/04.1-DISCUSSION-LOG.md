## Generation entry behavior

| Option | Description | Selected |

## Desktop-local lesson fidelity

| Option | Description | Selected |
|--------|-------------|----------|
| Full lesson fidelity | Bottle 1.0 desktop-local output must include usable cover, playable video binding, and normal history/learning behavior | ✓ |
| Subtitle-only acceptable for local path | Allow desktop-local to complete quickly with subtitles while leaving video/cover for later manual recovery | |
| Separate “local draft” lesson type | Treat desktop-local output as a temporary draft lesson distinct from normal history items | |

**User's direction:** Full lesson fidelity  
**Notes:** User reported that current Bottle 1.0 results are too shallow: upload page and history show no real cover, history requires rebinding, learning opens to black video, and the perceived output is “only subtitles”.

## Translation and output expectations

| Option | Description | Selected |
|--------|-------------|----------|
| Normal translated lesson contract | Bottle 1.0 desktop-local should still land in the same translated lesson contract as Bottle 2.0 unless an explicit partial state is shown | ✓ |
| Subtitle-only success by default | Treat untranslated subtitle output as the normal local-generation result | |
| Translation as a later manual action | Generate subtitles now and require a separate follow-up translation step | |

**User's direction:** Normal translated lesson contract  
**Notes:** User called out that Bottle 1.0 output currently appears to have “有字幕没翻译也没视频”, which is not acceptable as the default success case.

---

## Discussion summary

- The shared generate CTA should remain the only primary action.
- Model selection should drive generation routing.
- Bottle 1.0 local generation is only acceptable if the resulting lesson is immediately usable in history and learning.
- Partial local output must be surfaced honestly; it cannot masquerade as a full success.
