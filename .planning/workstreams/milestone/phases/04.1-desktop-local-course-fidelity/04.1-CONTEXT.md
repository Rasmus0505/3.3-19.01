# Phase 04.1: Desktop Local Course Fidelity - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Stabilize the desktop Bottle 1.0 learner-facing flow so a user can choose a model card, click the shared generate CTA, and receive a fully usable lesson with cover, playable local media binding, translated subtitle content, and consistent history / learning behavior. This phase covers model-card-to-runtime routing, desktop-local course save fidelity, and media / subtitle persistence for locally generated lessons. It does not redesign pricing, admin billing workflows, or broader post-v2 product polish outside the desktop local course path.

</domain>

<decisions>
## Implementation Decisions

### Entry behavior
- **D-01:** Upload should expose one primary generate CTA for both Bottle 1.0 and Bottle 2.0.
- **D-02:** The chosen model card, not a separate button, determines which generation path runs.
- **D-03:** Bottle 1.0 on desktop should route to the desktop-local generation pipeline when its runtime track is desktop-local.
- **D-04:** The UI must not show a dedicated `本地生成` button once the shared CTA can correctly dispatch by selected model/runtime.

### Desktop-local course fidelity
- **D-05:** A successful Bottle 1.0 desktop-local generation must produce a lesson that is learner-usable immediately, not merely a subtitle shell.
- **D-06:** Desktop-local generated lessons must persist playable media into client storage when `media_storage=client_indexeddb`.
- **D-07:** History cards for desktop-local lessons must show a real video cover when video media exists; `默认封面` is only acceptable when no cover can truly be derived.
- **D-08:** Learning view for desktop-local lessons must bind the saved local video automatically instead of landing in a black-screen / unbound state.

### Subtitle / translation behavior
- **D-09:** Bottle 1.0 desktop-local generation should still produce the same translated lesson contract expected by history and learning unless an explicit partial-translation state is returned.
- **D-10:** If translation genuinely fails or is deferred, the user-visible status must say so explicitly; do not silently present subtitle-only output as a full success.
- **D-11:** Local subtitle/media caches must support lesson identifiers from both server-backed numeric IDs and desktop-generated string IDs when those IDs enter the client persistence path.

### Failure behavior
- **D-12:** Shared-generate CTA dispatch must not misroute Bottle 1.0 desktop-local generation into a cloud failure message when the user intentionally selected the local model/runtime path.
- **D-13:** Packaged desktop helper builds must include all runtime assets required for actual local ASR execution, including VAD-related faster-whisper assets.

### the agent's Discretion
- Exact CTA copy once the dedicated `本地生成` button is removed
- Whether local-only artifacts should reuse existing persistence stores with broadened key handling or move to a new key namespace
- Exact learner-facing copy for partial translation vs full success

</decisions>

<specifics>
## Specific Ideas

- The selected model card should be the single source of truth for generation routing.
- `开始生成` should run Bottle 1.0 desktop-local when Bottle 1.0 is selected on desktop, and Bottle 2.0 cloud when Bottle 2.0 is selected.
- The desktop-local success path should save the chosen media file, derive a usable video cover, and immediately make the resulting lesson playable from history / learning.
- Local media and subtitle persistence should not assume lesson IDs are always positive integers if desktop-local creation can surface non-numeric identifiers in the client path.
- The product should clearly distinguish between:
  1. full success with translation and media,
  2. partial success with subtitles only,
  3. local generation failure.
- Packaging validation should include a real desktop-local transcription smoke check with a packaged helper/runtime, not only static contract checks.

</specifics>

## Deferred Ideas
