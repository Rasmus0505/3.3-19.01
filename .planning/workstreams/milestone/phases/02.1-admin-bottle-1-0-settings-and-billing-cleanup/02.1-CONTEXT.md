# Phase 2.1: Admin Bottle 1.0 Settings & Billing Cleanup - Context

**Gathered:** 2026-03-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Remove the dedicated admin "模型管理" surface and the separate Bottle 1.0 runtime-settings workflow, then fold Bottle 1.0 into the existing billing configuration flow as a normal billable model entry. This phase is about admin/billing cleanup and contract alignment only; it does not redesign learner-facing generation UX or change Phase 3/4 scope.

</domain>

<decisions>
## Implementation Decisions

### Admin surface restructuring
- **D-01:** Admin should no longer expose a standalone "模型管理" page or tab group.
- **D-02:** The current Bottle 1.0 dedicated settings tab is removed instead of being relocated as another standalone admin workspace.
- **D-03:** "计费与启停" remains the surviving admin surface for model-level operations in this scope.

### Bottle 1.0 runtime settings cleanup
- **D-04:** The dedicated Bottle 1.0 admin frontend and backend settings flow should be deleted end-to-end, including its standalone tab, fetch/save/rollback actions, and persistence contract.
- **D-05:** This phase removes the special-case admin control path for Bottle 1.0 instead of preserving partial dead code for later.
- **D-06:** Any remaining Bottle 1.0 operational behavior must keep working through existing desktop/runtime defaults rather than admin-edited Faster Whisper settings.

### Billing alignment
- **D-07:** Bottle 1.0 must be added into the existing billing configuration list as a normal model row with the same editing/save flow used by other billable models.
- **D-08:** Bottle 1.0 billing must hook into the same admin/public rate contracts, seed/default-rate flow, and model activation path as Bottle 2.0 rather than introducing a side channel.
- **D-09:** The change must preserve end-to-end model-name continuity so admin rate editing, public `/api/billing/rates`, and actual Bottle 1.0 generation/billing lookup all point at the same canonical model key.

### Scope guardrails
- **D-10:** This phase does not add new learner-facing Bottle 1.0 UX, new desktop runtime capabilities, or Phase 3 lesson-normalization work.
- **D-11:** Redeem-code, wallet, and unrelated admin workspaces stay unchanged except where navigation must drop the removed "模型管理" entry.

### the agent's Discretion
- Exact admin navigation placement after removing the old "模型管理" entry
- Whether Bottle 1.0 display text in billing uses "Bottle 1.0" or an existing localized label, as long as the canonical model key stays aligned
- Exact migration strategy for removing obsolete Faster Whisper admin persistence if the table or schema can remain safely unused

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `frontend/src/features/admin-rates/AdminRatesTab.jsx`: already loads `/api/admin/billing-rates`, edits drafts keyed by `model_name`, and saves each model through the shared PUT contract.
- `app/api/routers/admin.py`: already exposes the billing-rate admin endpoints and the dedicated Faster Whisper settings endpoints from one router, so this phase can consolidate within one backend surface.
- `app/api/routers/billing.py`: already returns public `rates` plus subtitle settings; adding Bottle 1.0 to the billing list should flow through this contract automatically if seeded/listed correctly.
- `app/models/billing.py`: `BillingModelRate` already has the status, concurrency, segmentation, and price fields needed for Bottle 1.0 billing/admin control.

### Established Patterns
- Admin model-level editing is keyed by canonical `model_name` and saved per row through `/api/admin/billing-rates/{model_name}`.
- Public billing data is sourced from the same backend billing service layer rather than a separate admin-only configuration store.
- Bottle 1.0 desktop generation is already shipped; this phase should align admin/billing contracts around it, not rebuild the generation pipeline.

### Integration Points
- Admin frontend removal will touch the admin router/navigation that currently lands on `AdminModelsPage`.
- Billing continuity depends on aligning `BillingModelRate`, admin rate listing, public rate listing, and whichever runtime/service code resolves Bottle 1.0 billing by model key.
- Removing Faster Whisper admin settings will require deleting dead frontend imports/routes and backend endpoints without breaking unrelated admin workspaces.

</code_context>

<specifics>
## Specific Ideas

- The dedicated Bottle 1.0 admin settings flow should disappear completely instead of being hidden behind another tab.
- Bottle 1.0 should show up in the same billing configuration table as the other models so pricing, activation, and future billing behavior are maintained in one place.
- The highest-risk part of this phase is model-key continuity: adding Bottle 1.0 to billing must line up with the actual runtime/billing lookup path, not just render a new admin row.

</specifics>
