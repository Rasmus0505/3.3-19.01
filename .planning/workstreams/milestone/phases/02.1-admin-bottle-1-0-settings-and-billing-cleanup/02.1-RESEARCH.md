# Phase 2.1: Admin Bottle 1.0 Settings & Billing Cleanup - Research

**Researched:** 2026-03-27
**Domain:** FastAPI backend billing service + React admin frontend (deletion + billing integration)
**Confidence:** HIGH

## Summary

This phase is a deletion + billing-alignment cleanup. The `AdminModelsPage` ("цибхЮЛчобчРЖ") with its two tabs (`billing` + `faster-whisper`) must be removed from the admin surface, and `faster-whisper-medium` (Bottle 1.0) must be added to the existing billing rate list so it appears in `AdminRatesTab` and the public `/api/billing/rates` endpoint.

The canonical model key `FASTER_WHISPER_ASR_MODEL = "faster-whisper-medium"` is already used in:
- `app/services/asr_model_registry.py` тАФ display metadata ("Bottle 1.0", "desktop_local")
- `app/services/billing.py` тАФ `DEFAULT_MODEL_RATES` seeds it with default values
- `app/services/faster_whisper_asr.py` тАФ runtime transcription logic
- `app/repositories/billing_rates.py` тАФ billing lookup
- `app/serializers.py` тАФ `to_rate_item()` with correct display_name

The **only missing piece** is that `FASTER_WHISPER_ASR_MODEL` is absent from `ADMIN_BILLING_MODEL_ORDER` in `app/services/billing.py`, so `list_admin_rates()` never returns it тАФ and thus `AdminRatesTab` never shows it. Adding it to that tuple closes the loop end-to-end without any schema or API contract changes.

**Primary recommendation:** Add `"faster-whisper-medium"` to `ADMIN_BILLING_MODEL_ORDER`, delete the 4 faster-whisper-settings backend endpoints and the 3 frontend files, then wire the nav redirect.

### Pattern 1: Billing Rate Admin тЖТ Public Contract

**What:** `AdminRatesTab` (frontend) calls `/api/admin/billing-rates` (GET) тЖТ `list_admin_rates()` (service) тЖТ `BillingModelRate` table. `PublicRatesTab` (or any client) calls `/api/billing/rates` (GET) тЖТ `list_public_rates()` (service) тЖТ same table with `is_active=True` filter.

**Flow for this phase:**
1. `AdminRatesTab` GETs `/api/admin/billing-rates` тЖТ returns all rates in `ADMIN_BILLING_MODEL_ORDER`
2. `AdminRatesTab` PUTs `/api/admin/billing-rates/{model_name}` тЖТ updates `BillingModelRate` row
3. Public endpoint GETs `/api/billing/rates` тЖТ returns active rates in `PUBLIC_BILLING_MODEL_ORDER` (includes `FASTER_WHISPER_ASR_MODEL` already)
4. Runtime `get_model_rate(db, "faster-whisper-medium")` тЖТ returns `BillingModelRate` for billing lookup

**No new endpoints needed.** The contract is already in place.

### Pattern 2: Admin Operation Log Audit Trail

The `append_admin_operation_log()` pattern in `admin.py` is used for:
- `faster_whisper_settings_update` / `faster_whisper_settings_rollback` тАФ these log entries will remain in the DB but become orphaned. No migration needed; they are inert.
- `billing_rate_update` тАФ existing pattern, no change

### Pattern 3: Runtime Settings vs. Admin Billing

- **Runtime settings** (device, compute_type, beam_size, etc.) come from `FasterWhisperSetting` DB table (backend) or desktop defaults (`_DEFAULT_SETTINGS` in `faster_whisper_asr.py`). Admin no longer edits these.
- **Billing** (price, cost, is_active, concurrency) comes from `BillingModelRate` table. Admin edits these via `AdminRatesTab`.
- D-06: Runtime behavior continues via desktop defaults тАФ no change needed in `faster_whisper_asr.py`.

### Anti-Patterns to Avoid
- **Don't re-add a standalone "Bottle 1.0 Settings" tab** тАФ D-01/D-02 explicitly forbid this.
- **Don't create a new billing endpoint for Bottle 1.0** тАФ use the existing `/api/admin/billing-rates` and `/api/billing/rates` contracts (D-08).
- **Don't change the canonical model key** тАФ `"faster-whisper-medium"` must remain consistent across all layers (D-09).
- **Don't touch unrelated admin workspaces** тАФ redeeming, wallet, health, security, users pages stay untouched (D-11).

## Common Pitfalls

### Pitfall 1: Orphaned backend endpoints returning 500
**What goes wrong:** After deleting the frontend `AdminModelsPage`, the backend endpoints `/api/admin/faster-whisper-settings*` still exist. If any client still holds old URLs or the desktop helper has stale references, those calls return 500 (auth check passes, model imports may fail).
**Why it happens:** The 4 faster-whisper-settings endpoints in `admin.py` are not deleted as part of the frontend-only cleanup.
**How to avoid:** Delete the 4 endpoints: `admin_get_faster_whisper_settings`, `admin_get_faster_whisper_settings_history`, `admin_update_faster_whisper_settings`, `admin_rollback_faster_whisper_settings_last` from `admin.py`.
**Warning signs:** Backend logs showing `404` or `500` on `/api/admin/faster-whisper-settings*` after deployment.

### Pitfall 2: Bottle 1.0 not appearing in AdminRatesTab
**What goes wrong:** The `AdminRatesTab` GET `/api/admin/billing-rates` returns only the models in `ADMIN_BILLING_MODEL_ORDER`. Since `FASTER_WHISPER_ASR_MODEL` is not in that tuple, Bottle 1.0 is silently absent.
**Why it happens:** `list_admin_rates()` filters: `row.model_name in {ADMIN_BILLING_MODEL_ORDER}`. If `"faster-whisper-medium"` is missing, it's excluded.
**How to avoid:** Add `FASTER_WHISPER_ASR_MODEL` to `ADMIN_BILLING_MODEL_ORDER` in `billing.py`. This is the single-line fix that closes D-07.
**Warning signs:** Admin sees only 2 rows (Bottle 2.0 + MT) in the billing table instead of 3.

### Pitfall 3: Stale imports breaking the admin app build
**What goes wrong:** `AdminModelsPage.jsx` imports `AdminFasterWhisperSettingsTab`. After deleting that file, the build fails because the import resolves to a missing module.
**Why it happens:** `AdminModelsPage` is imported in `AdminApp.jsx`. Deleting the page without removing the import chain breaks the frontend bundle.
**How to avoid:** Delete all three files together: `AdminModelsPage.jsx`, `AdminFasterWhisperSettingsTab.jsx`, and remove the `models` route from `AdminApp.jsx` in the same commit.

### Pitfall 4: Navigation dead-end after route deletion
**What goes wrong:** If the user navigates to `/admin/models` after the route is removed, they get a `*` wildcard redirect to `health`. But the `rates` тЖТ `models?tab=billing` redirect still works; the `subtitle-settings` тЖТ `models` redirect may become confusing.
**Why it happens:** Legacy redirects in `AdminApp.jsx` point to the now-deleted `models` route.
**How to avoid:** Update the redirect routes to point to a sensible default (e.g., `health` or remove them if no longer needed per D-11).
