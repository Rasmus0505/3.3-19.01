## Phase-Specific UI Contracts

### D-01 / D-02 / D-03 — Navigation Restructuring

- `ADMIN_NAV_ITEMS` in `adminSearchParams.js`: Remove entry with `key: "models"`, `label: "模型管理"`
- `resolveAdminNavKey` in `adminSearchParams.js`: Remove `/admin/models` branch
- `AdminApp.jsx`: Remove `<Route path="models" ... />`
- Admin redirects: `/admin/rates` and `/admin/subtitle-settings` already redirect to `/admin/models?tab=billing` — confirm these continue to redirect to `/admin/health` or equivalent after removal
- "计费与启停" tab (`billing` value in `MODEL_TABS`) survives as-is; no tab removal needed for billing tab

### D-04 / D-05 / D-06 — Bottle 1.0 Settings Deletion

Files to remove from admin frontend routing:
- `frontend/src/features/admin-faster-whisper-settings/AdminFasterWhisperSettingsTab.jsx`
- `frontend/src/features/admin-pages/AdminModelsPage.jsx` (entire file, or remove `faster-whisper` tab entry)
- Import of `AdminFasterWhisperSettingsTab` from `AdminModelsPage.jsx`

Backend files referenced for deletion (UI contract only; backend contracts tracked separately):
- Remove `AdminFasterWhisperSettingsTab` import from admin router if any
- No new UI components needed

### D-07 / D-08 / D-09 — Bottle 1.0 in Billing Table

- Bottle 1.0 (`faster-whisper` model key) must appear as a row in `AdminRatesTab` billing table
- No changes to `AdminRatesTab` rendering — the component already iterates all rates from `/api/admin/billing-rates`
- Billing unit for Bottle 1.0: `minute` (ASR model, not token-based MT)
- `display_name` in the row: use `display_name` field if set, fall back to canonical key `faster-whisper`
- `is_active` switch in the row controls Bottle 1.0 activation, matching Bottle 2.0 behavior
- No new UI patterns — Bottle 1.0 follows identical row layout as existing models

### D-11 — Navigation Update for Removed Entry

- Admin health page remains at `/admin/health` (no change)
- "计费与启停" accessible at `/admin/models?tab=billing` — confirm `AdminApp` still mounts `AdminModelsPage` OR consider direct billing route
- Recommend: Add a `/admin/rates` route pointing directly to the billing table, since `AdminModelsPage` with only one tab may be redundant
  - Option A: Keep `AdminModelsPage` with single-tab (removes `MODEL_TABS` wrapper, renders `AdminRatesTab` directly)
  - Option B: Create `/admin/rates` page component rendering `AdminRatesTab` directly
  - Executor discretion: Either approach is valid
