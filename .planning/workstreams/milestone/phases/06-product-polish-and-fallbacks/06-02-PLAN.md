<objective>
Improve the desktop billing UX in the upload panel so insufficient billing is handled as a recoverable state with a clear recovery path, and the estimated charge display is simplified to reduce UI noise in the idle state.

Purpose: Users with insufficient balance see an actionable path to recharge rather than a confusingly disabled button, and the estimated-price display no longer clutters the UI with ASR/MT breakdown tooltips.
</objective>
