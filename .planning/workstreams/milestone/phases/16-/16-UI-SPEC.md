- Delete confirmation button in admin announcement list
- Announcement deletion confirmation dialog

**Note:** `--upload-brand` is the established accent blue for Bottle brand. Reuse it for announcement CTAs and active states.

## Admin Announcement Page

### Layout
- New tab page under admin shell: `/admin/announcements`
- Tab structure via `Tabs` component
- Left: announcement list (scrollable `ScrollArea`)
- Right: announcement editor panel (or empty state)

### Announcement List Item (Card)
- Title (18px semibold) — truncated at 1 line with ellipsis
- Type badge (`Badge` component, colored by type)
- Status: active (green dot) / inactive (gray)
- Created date (muted text, 13px)
- Hover: subtle `--secondary` background
- Actions: edit button, delete button (right-aligned)

### Announcement Editor
- Title input (`Input` component)
- Type selector (`Select`: changelog / banner / modal — use Chinese labels)
- Content textarea (`Textarea` component, Markdown supported)
- Status toggle (`Switch` — active/inactive)
- Action buttons: "保存" (primary, `bg-upload-brand`), "取消" (secondary)
- Delete button at bottom (destructive, only for existing announcements)

### Empty State (Admin)
- Heading: "暂无公告"
- Body: "点击上方按钮创建第一条公告"
- Icon: bell or announcement icon from lucide-react

## User-Facing: Banner Type

### Placement
- Full-width banner at top of page, below nav bar
- Appears on login/app open for active banner announcements
- Fixed position (does not scroll)

### Structure
- Left border: 4px solid `--upload-brand`
- Background: `--secondary: oklch(0.97 0 0)`
- Padding: 12px 16px
- Layout: `[close X] [content area]`
- Content: title (16px semibold) + body text (14px regular)
- Close button: top-right, X icon from lucide-react, 24x24px touch target

### Close Behavior
- User clicks X → banner disappears (CSS transition: slide up 200ms)
- Closed state stored in component state (session-only)
- No localStorage persistence

### Dismissing Per Banner
- Each banner announcement dismissed independently
- Closing one banner does not affect other banner announcements

---

## User-Facing: Modal Type

### Placement
- Centered overlay dialog via `Dialog` component
- Backdrop: semi-transparent dark overlay
- Appears immediately on login/app open for active modal announcements

### Structure
- Width: max-w-md (448px)
- Header: announcement title (18px semibold) + close X button
- Body: content with Markdown rendering support
- Footer: "我已知晓" button (`bg-upload-brand` primary CTA)
- Border radius: `--radius-lg` (matches shadcn default)

### Close Behavior
- User clicks "我已知晓" → modal closes
- User clicks X → modal closes
- Closed state stored in component state (session-only)
- No localStorage persistence

### Multiple Modals
- If multiple modal announcements active, show one at a time (queue)
- After dismissing one, show next modal
- After all dismissed, session is clear

---

## User-Facing: Changelog Type

### Placement
- Embedded section in personal center / settings page
- Existing "个人中心" or "设置" page expanded with announcement section

### Structure
- Section heading: "更新日志" (24px semibold)
- Chronological list (newest first)
- Each item: date (muted, 13px) + title (16px semibold) + type badge
- Click to expand: shows full content (Markdown rendered)
- Collapsed by default, expanded on click
- Scrollable list if many items (`ScrollArea`)

### Empty State
- Heading: "暂无更新日志"
- Body: "关注 Bottle 最新动态，不错过任何新功能"
