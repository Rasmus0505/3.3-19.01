## Phase Overview

生词本管理功能增强，包括批量操作、局部翻译、UI 风格统一、轻提示系统。

---

## Goals (from Requirements.md / Project Overview)

1. **批量操作**: 支持多选生词、批量删除、批量归档、批量移动到其他列表
2. **局部翻译**: 用户选中单词/短语后调用翻译 API 获取翻译结果
3. **UI 风格统一**: 列表视图采用 shadcn 风格，与 Phase 17 复习视图保持一致
4. **轻提示系统**: 为关键交互点提供半透明悬浮说明

## Decisions

| # | Decision | Rationale |
|---|----------|------------|
| D-18-01 | 批量选择使用**混合模式**（Checkbox + Shift 多选） | 兼顾小白用户（Checkbox）和高级用户（Shift 多选） |
| D-18-02 | 批量操作使用**顶部悬浮工具栏** | 选中 N 项后顶部出现操作按钮，不占用列表空间 |
| D-18-03 | 局部翻译是**调用翻译 API 重新获取**，非查看已存储翻译 | 用户选中单词/短语后实时调用 API |
| D-18-04 | 新建**轻提示组件**（半透明悬浮样式） | "半透明悬浮"是明确视觉需求，需要定制样式 |
| D-18-05 | 轻提示触发：**Hover** | 鼠标悬浮时显示 |
| D-18-06 | **沉浸式学习按钮**都加上轻提示：Hover + 点击后各显示一次 | 确保用户点击后仍能理解按钮含义 |
| D-18-07 | shadcn 风格收口范围：**列表 + 复习视图** | 与 Phase 17 UI-SPEC 对齐，支持管理/复习视图切换 |
| D-18-08 | 局部翻译 API 使用 **`qwen-mt-flash`** 模型 | 效果好、速度快、成本低、支持流式输出 |
| D-18-09 | 用户选中单词/短语后**实时调用 API**，展示翻译弹窗 | 即时反馈，低成本 |

## API Cost Analysis (MT 翻译模型)

### 计费方式
- **按 Token 计费**：input_tokens + output_tokens
- **单价固定**，与调用次数无关
- 不存在"批量调用打折"机制

### 少量多次 vs 一次大量
| 方式 | Token 消耗 | 总成本 |
|------|-----------|--------|
| 少量多次 | 10 × (10 input + 5 output) = 150 | **相同** |
| 一次大量 | 10 input + 5 output = 55 | **相同** |

**结论：成本完全相同**

### 实际差异
| 因素 | 少量多次 | 一次大量 |
|------|---------|---------|
| 响应延迟 | 碎片化等待 | 一次性等待 |
| 网络开销 | 多次 HTTP | 一次 HTTP |
| 用户交互 | 可中断 | 需等全部完成 |

### 推荐
- **短文本（单词/短语）**：每次调用（实时反馈）
- **长文本（句子/段落）**：批量调用
