# Phase 14 Plan .1: Gap Closure — Version Card + Auto-Check Summary

## Objective

Close Gap 1 (version display card partial) and Gap 2 (auto-check on launch conditional) from `14-VERIFICATION.md`.

**Gap 1:** UploadPanel diagnostics showed a version badge but NOT a dedicated side-by-side version card with both `localVersion` AND `remoteVersion` when `updateAvailable` is true.

**Gap 2:** Bootstrap auto-check (main.mjs line 1266) was guarded by `desktopRuntimeConfig?.clientUpdate?.checkOnLaunch !== false` but write-runtime-defaults.mjs may not always set this field.

## What Was Done

### Task 1: Add dedicated blue version card with localVersion AND remoteVersion side-by-side

**Status:** Already implemented — verified present in `UploadPanel.jsx` at lines 7295-7343.

The version card was found as the first child of the diagnostics dialog `<div className="space-y-4">`, before the `desktopDiagnosticsItems` grid. It includes:

- `border-blue-200 bg-blue-50` blue card container
- "桌面客户端版本" card title
- Left panel: "当前版本" label with `localVersion` or `currentVersion`
- Right panel: "可更新至" with `remoteVersion` in green when `updateAvailable === true`
- "已是最新版本" in green when `updateAvailable === false` and not checking
- "检查中..." when status is "checking"
- "检查更新" button calling `checkClientUpdate`
- "立即更新" button calling `startClientUpdateDownload` (only when updateAvailable)

**Verification:** `grep -n "桌面客户端版本\|当前版本\|可更新至\|已是最新版本\|border-blue-200" frontend/src/features/upload/UploadPanel.jsx` returns 6+ matches including the version card at lines 7295-7343.

### Task 2: Ensure explicit checkOnLaunch: true default in write-runtime-defaults.mjs

**Status:** Already implemented — verified present in `write-runtime-defaults.mjs` at line 30.

The `checkOnLaunch` field uses `|| "true"` fallback:

```javascript
checkOnLaunch: !["0", "false", "no", "off"].includes(
  String(process.env.DESKTOP_CLIENT_UPDATE_CHECK_ON_LAUNCH || "true").trim().toLowerCase()
),
```

This means:
- If env var is not set → defaults to `"true"` → `checkOnLaunch = true` (auto-check fires)
- If env var is `"false"`/`"0"`/`"no"`/`"off"` → `checkOnLaunch = false`
- Otherwise, env var value is used

main.mjs line 1266 correctly reads it via `desktopRuntimeConfig?.clientUpdate?.checkOnLaunch`.

**Verification:** Both grep patterns returned matches confirming the field name is consistent between both files.
