# English Sentence Spelling Trainer — 产品介绍

> 本文档以**业务单元为视角**，描述产品的功能全景与设计意图。
> 产品代号：Bottle，可作为 Web 服务（Zeabur 部署）或 Windows 桌面客户端（Electron 打包）运行。

---

## 一、业务总览

| 业务单元 | 核心动作 | 用户角色 |
|---------|---------|---------|
| 认证与账户 | 注册、登录、登出 | 全部用户 |
| 素材管理 | 上传音视频 | 普通用户 |
| 课程生成 | AI 自动生成练习课程 | 普通用户（后端自动触发） |
| 学习练习 | 逐句跟读、拼写检查 | 普通用户 |
| 词汇积累 | 收藏单词、管理单词本 | 普通用户 |
| 钱包与充值 | 余额管理、兑换码充值 | 普通用户 |
| 运营管理 | 用户、计费、兑换码、系统监控 | 管理员 |

## 二、业务单元详述

### 业务 1：认证与账户

**目标**：识别用户身份，建立会话，区分普通用户与管理员。

#### 用户旅程

1. 未注册用户 → 输入邮箱、密码 → 创建账户 → 自动登录
2. 已注册用户 → 输入邮箱、密码 → 登录系统
3. 已登录用户 → 点击登出 → 清除会话 → 返回登录状态

#### 技术实现

- **认证机制**：JWT（JSON Web Token）
- **Token 策略**：
  - `access_token`：有效期 15 分钟，存前端内存（不落磁盘）
  - `refresh_token`：有效期 7 天，写入 HTTP-only Cookie（防 XSS）
- **刷新机制**：access_token 过期前静默刷新，用户无感知
- **会话管理**：无状态，服务端不存会话；容器重启不影响已有会话

#### 涉及数据

| 数据表 | 操作 |
|--------|------|
| `users` | 创建账户、验证密码、更新最后登录时间 |
| `user_login_events` | 记录登录时间、IP 地址 |

#### 角色体系

- **普通用户**：注册登录后自动获得
- **管理员**：由管理员手动指定，可提升其他用户为管理员

#### 设计意图

- JWT 无状态，适合 Zeabur 容器化横向扩缩
- HTTP-only Cookie 防 refresh_token 被 XSS 盗取
- 不存服务端会话，容器重启零代价

### 业务 2：素材管理

**目标**：接收用户上传的音视频文件，持久化存储，供后续 ASR 处理。
文件本身不经过应用服务器，由客户端直传到 DashScope 对象存储。

#### 用户旅程

1. 进入上传界面（侧边栏"上传"按钮）
2. 拖拽或点击选择文件（支持 mp4, mp3, wav, m4a, webm, avi, mkv, mov）
3. 确认文件名和大小
4. 点击"开始生成" → 客户端直传到 DashScope（不经过应用服务器）
5. 系统自动进入课程生成流程；用户可离开页面

#### 技术实现

> **Bottle 1.0**（本地 faster-whisper）：
> - 视频文件由客户端直接上传到 DashScope OSS
> - 后端仅保存 DashScope 返回的文件 URL / file_id
> - 后续由本地 faster-whisper 处理

> **Bottle 2.0**（DashScope 直传 + 云端 ASR）：
> - 视频文件由客户端直接上传到 DashScope OSS
> - 后端仅保存 DashScope 返回的文件 URL / file_id
> - 后续由 DashScope 云端 ASR 直接读取文件处理

#### 涉及数据

| 数据表 | 操作 |
|--------|------|
| `media_assets` | 创建媒体资产记录（DashScope URL / file_id、时长、格式） |
| `lesson_generation_tasks` | 创建任务记录，状态为 PENDING |

#### 设计意图

- **视频直传**：文件不经过应用服务器，保护用户隐私 + 节省服务器带宽
- **极简操作**：选文件 → 点按钮 → 等待结果
- 用户上传后可离开，无需保持页面

### 业务 3：课程生成

**目标**：把原始音视频转换为结构化的英文跟读练习课程（英文原文 + 中文翻译 + 时间戳音频片段）。

#### 内部流水线（用户不可见，后端自动执行）

> **Bottle 1.0**（本地 faster-whisper，文件不上传）：
> - 视频由客户端直传到 DashScope
> - 后端通过 DashScope file_id 让本地 faster-whisper 直接读取并处理
> - 处理完成后，结果写入 PostgreSQL

> **Bottle 2.0**（DashScope 直传 + 云端 ASR）：
> - 视频由客户端直传到 DashScope
> - 后端通过 DashScope file_id 让云端 ASR 直接读取并处理
> - 处理完成后，结果写入 PostgreSQL

```
视频上传完成
      ↓
[阶段1] UPLOADING
  客户端直传到 DashScope OSS
  保存 file_id（文件不经过应用服务器）
      ↓
[阶段2] TRANSCRIBING
  ASR 转写（两套引擎可选）：
    - Bottle 1.0：本地 faster-whisper-medium，通过 file_id 读取
    - Bottle 2.0：DashScope 云端 qwen3-asr-flash-filetrans
  输出：带时间戳的词列表（word, start, end）
      ↓
[阶段3] SEGMENTING
  按语义或固定 token 数切分句子
  每个句子分配起始/结束时间戳
      ↓
[阶段4] TRANSLATING
  批量调用 qwen-mt-flash 翻译
  英文原文 → 中文翻译
      ↓
[阶段5] FINALIZING
  生成 Lesson 记录
  生成 LessonSentence 记录（每句一条，含音频片段元数据）
  扣减用户钱包余额（按分钟费率和 token 费率计费）
      ↓
[阶段6] COMPLETED
  课程就绪，通知前端
```

#### 字幕变体

- 同一课程可存储多条翻译（不同翻译策略：直译、意译等）
- 用户在练习时可切换字幕变体

#### 桌面客户端（Electron）

- Bottle 1.0 可通过 Electron 桌面客户端运行
- 本地 Python 子进程运行 faster-whisper，文件不上传到云端（客户端直传到 DashScope，后端通过 file_id 读取）
- 适合：隐私敏感用户、无稳定外网的环境、想省云端 ASR 费用的用户

#### 涉及数据

| 数据表 | 操作 |
|--------|------|
| `lesson_generation_tasks` | 创建并更新任务状态（PENDING→UPLOADING→TRANSCRIBING→SEGMENTING→TRANSLATING→FINALIZING→COMPLETED） |
| `lessons` | 创建课程记录 |
| `lesson_sentences` | 批量创建句子记录（英文原文、中文翻译、时间戳、token 列表） |
| `media_assets` | 关联媒体资产（存储 DashScope file_id） |
| `wallet_ledger` | 扣费流水记录（TRANSLATION 类型） |
| `subtitle_settings` | 读取 ASR 模型、分词策略、翻译批次大小配置 |

#### 设计意图

- 阶段拆分：每个阶段独立状态，便于定位错误、单阶段重试、监控进度
- **视频直传保护隐私 + 节省服务器带宽**：Bottle 1.0 和 Bottle 2.0 均由客户端直传 DashScope，文件不经过应用服务器
- 两套 ASR 引擎：云端省事、本地省钱 + 保护隐私
- 翻译扣费：按 token 计费，精确计量
- 字幕变体：支持不同翻译风格，满足不同学习偏好

### 业务 4：学习练习

**目标**：用户通过"听 → 理解 → 输入 → 即时反馈"的闭环，练习英文听力与拼写。

#### 练习模式 A：普通练习

1. 用户从历史课程列表点进某个课程
2. 系统加载该课程所有句子（按生成顺序排列）
3. **逐句循环**：
   - 自动播放当前句子对应的时间段音频（只播这一句）
   - 显示中文翻译（参考）
   - 用户在输入框输入英文原文
   - **实时逐 Token 拼写检查**：输入时即高亮正确/错误字符
   - 用户按回车提交 → 判断整句是否正确 → 正确则进入下一句
   - 进度自动保存（用户可中途关闭页面，下次继续）
4. 可选操作：跳过句子、查看参考原文（折叠隐藏防止直接抄）、切换字幕变体

#### 练习模式 B：沉浸式练习

- 从普通练习中点击"沉浸模式"进入
- 全屏界面：只展示中文翻译 + 音频控件 + 输入框
- 极简 HUD：进度条（X/Y）、退出按钮
- 提交正确后 1.5 秒自动进入下一句，无需手动点击
- 目标：消除视觉干扰，专注跟读训练

#### 涉及数据

| 数据表 | 操作 |
|--------|------|
| `lessons` | 读取课程信息（标题、句子列表） |
| `lesson_sentences` | 读取句子（英文原文、中文翻译、音频时间戳） |
| `lesson_progress` | 读取/写入用户进度（当前句子索引、已完成句子索引） |

#### 设计意图

- **逐 Token 检查**：比整句对错反馈更有教学价值——用户知道具体哪个词拼错了
- **音频片段播放**：只播放当前句子，不是整段音频，避免信息过载
- **进度自动保存**：练习可中断、可续接，降低用户心理负担
- **沉浸模式**：减少视觉噪音，模拟真实跟读场景

### 业务 5：词汇积累

**目标**：用户在练习过程中或练习完成后，将不熟悉的单词单独收藏到单词本，供后续复习。

#### 用户旅程

1. 在练习界面（普通或沉浸式），点击某个单词
2. 单词被收藏到单词本（关联当前课程和句子来源）
3. 进入单词本页面，浏览所有收藏的单词
4. 可按单词搜索、按来源课程筛选、删除不需要的单词
5. 点击单词可跳转到来源句子重新练习

#### 涉及数据

| 数据表 | 操作 |
|--------|------|
| `wordbook_entries` | 创建（收藏单词）、读取（列表）、删除（清理） |
| `wordbook_entry_sources` | 记录单词与 LessonSentence 的关联关系 |

#### 设计意图

- 词汇积累是独立于课程的长线需求：课程可以复习完，但单词需要反复记忆
- 收藏动作要轻：练习过程中一键收藏，不打断练习流
- 支持溯源：知道单词来自哪句话，可快速跳转复习

---

### 业务 6：钱包与充值

**目标**：管理用户账户余额，支持用户为课程生成付费。

#### 余额管理

- 用户有且只有一个钱包账户
- 侧边栏实时显示余额（余额不足时在上传和练习入口给出提示）
- 余额来源：兑换码充值（目前唯一渠道）

#### 兑换码充值

1. 用户点击余额徽章 → 进入兑换码充值面板
2. 输入兑换码
3. 系统验证：
   - 兑换码存在
   - 兑换码未被使用
   - 兑换码所属批次未过期
   - 兑换码所属批次未达使用上限
   - 该用户此前未使用过此兑换码（每码一人一次）
4. 验证通过 → 钱包余额增加 → 记录充值流水

#### 涉及数据

| 数据表 | 操作 |
|--------|------|
| `wallet_accounts` | 读取余额、增加余额 |
| `wallet_ledger` | 写入充值流水（REDEEM 类型） |
| `redeem_code_batches` | 验证批次状态（有效期内、人数未满） |
| `redeem_codes` | 验证单码状态（未使用、未被当前用户使用） |
| `redeem_code_attempts` | 记录每次兑换尝试（成功/失败） |

#### 设计意图

- 国内用户无法绑卡，兑换码是唯一充值渠道
- 批次 + 单码二级管控：批次定义规则，单码追踪使用记录
- 每码一人一次：防止一个码被反复充值
- 记录所有尝试：便于审计和排查问题

### 业务 7：运营管理

**目标**：管理员对用户、计费、兑换码、系统配置进行全面管控。

#### 子业务 7.1：用户管理

- 查看所有注册用户（邮箱、注册时间、余额、课程数、是否为管理员）
- 搜索 / 筛选用户
- 管理操作：提升为管理员、禁用账户

#### 子业务 7.2：安全监控

- 查看用户登录事件日志（UserLoginEvent）
- 可选：检测异地登录、频繁登录尝试等异常模式

#### 子业务 7.3：计费配置

- 配置分钟费率、Token 费率（实时生效）
- 配置后影响后续所有课程生成的扣费计算

#### 子业务 7.4：字幕生成配置

- 配置默认 ASR 模型（本地 faster-whisper / DashScope 云端）
- 配置分词策略（按 token 数或语义）
- 配置翻译批次大小
- 影响后续所有课程的生成过程

#### 子业务 7.5：兑换码管理

- **批次管理**：创建批次（面额、数量、有效期）→ 自动生成该批次下所有单码
- **单码管理**：激活、禁用、查看单个兑换码的使用记录
- **使用审计**：查看兑换码使用流水

#### 子业务 7.6：ASR 模型管理

- 本地 faster-whisper 模型：配置模型路径、是否启用
- DashScope 云端模型：选择具体模型名称

#### 子业务 7.7：运营概览

- 系统健康状态（数据库连接、外部 API 可用性）
- 关键运营指标（注册用户数、活跃用户、课程数、系统消耗）

#### 子业务 7.8：SQL 控制台

- 只读 SQL 查询界面
- 分页展示查询结果
- 防止误写操作

#### 涉及数据

| 数据表 | 操作 |
|--------|------|
| `users` | 读取用户列表、管理员权限变更 |
| `user_login_events` | 读取登录日志 |
| `billing_model_rates` | 读取/写入费率配置 |
| `subtitle_settings` | 读取/写入字幕生成配置 |
| `redeem_code_batches` | 创建批次、读取批次列表 |
| `redeem_codes` | 读取单码、变更状态 |
| `redeem_code_attempts` | 读取兑换记录 |
| `admin_operation_logs` | 记录管理员操作（配置变更、用户变更等） |
| `translation_request_logs` | 读取翻译请求日志（排查 ASR/翻译问题） |
| `faster_whisper_settings` | 读取/写入 faster-whisper 模型配置 |

#### 设计意图

- 计费配置实时生效：不需要重启服务即可调整费率
- 字幕配置统一管理：避免不同课程生成参数不一致
- 兑换码批次制：批量生成效率高，批次维度统计方便
- 操作日志全记录：管理员操作可追溯
- SQL 控制台只读：防止线上数据被误写

## 五、技术约束与边界

| 约束 | 说明 |
|------|------|
| 生产数据库 | PostgreSQL，禁止 SQLite |
| 部署平台 | Zeabur（GitHub 自动构建） |
| 前端 | React + Vite，构建产物同步到 `app/static/` |
| JWT | 无状态，适合容器化 |
| 媒体文件存储 | DashScope OSS（客户端直传，不经过应用服务器） |
| 敏感配置 | 环境变量，不进代码库 |

## 六、设计意图汇总

| 设计决策 | 原因 |
|---------|------|
