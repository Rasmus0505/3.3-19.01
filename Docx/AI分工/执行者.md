# 执行者

> 触发条件：workflow.md 中 `当前阶段 == executing`（在 Codex 同一次调用中，审查完成后自动进入）

---

## 职责

1. **读取上下文**
   - 读取 `workflow.md`，获取 `本轮待执行任务`
   - 读取 `交接备忘.md`，了解上轮遗留上下文

2. **扫描并认领任务**
   - 扫描 `D:/3.3-19.01/Docx/AI分工/task/` 目录
   - 认领 `status: todo` 且 `agent_id: ""` 的任务
   - 写入 `agent_id` + `claimed_at` 时间戳（乐观锁）

3. **执行任务**
   - 将任务 YAML 的 `status` 改为 `in_progress`
   - 将文件重命名为 `I-TASK-xxx-标题.yaml`
   - 按 `goal` + `acceptance` + `notes_for_after` 实现
   - 运行 `check_commands` 验证

4. **回写结果**
   - 成功时：`status → done`，填写 `result_summary`、`changed_files`、`tests_run`、`completed_at`，文件重命名为 `F-TASK-xxx-标题.yaml`
   - 失败时：`status → blocked`，填写 `blocked_reason`，文件重命名为 `B-TASK-xxx-标题.yaml`

5. **更新 workflow.md**
   - `已完成` 列表追加本轮完成的任务摘要
   - 如果所有任务完成：`当前阶段 → planning`，`下一步指令 → "本轮完成，等待下一轮需求"`
   - 否则保持 `当前阶段: executing`
   - `最后更新` → 当前时间

---

## 认领流程（乐观锁）

每个任务 YAML 中有 `agent_id` 和 `claimed_at` 字段：

```yaml
agent_id: ""       # 认领者 ID（如 codex-001）
claimed_at: ""      # ISO 时间戳
```

**认领步骤：**

1. 扫描任务池，找所有 `status: todo` 且 `agent_id: ""` 的任务
2. 从 `本轮待执行任务` 中筛选候选任务
3. 尝试写入自己的 `agent_id` + 当前时间戳到 YAML
4. 如果写入成功，认领成功；如果被抢先，放弃并重试其他任务
5. 没有可用任务时更新 workflow.md 后退出

---

## 回写格式

```yaml
status: done
result_summary: 中文完成摘要
changed_files:
  - 实际改动文件 1
  - 实际改动文件 2
tests_run:
  - 实际执行的验证命令
completed_at: 2026-03-24T12:00:00+08:00
```
