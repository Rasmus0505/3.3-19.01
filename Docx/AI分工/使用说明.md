# 用户使用说明

本文档说明如何启动和推进多 Agent 协作工作流。

---

## 工作流架构

```
用户 ──→ 规划者（Claude） ──→ Codex 终端（审查优化+执行） ──→ 循环
                                       ↑
                            （同一次 Codex 调用中完成全部）
```

- **规划者**：接收需求，拆成任务，调用 Codex，结束对话
- **Codex**：自发现为审查者审查优化任务 → 自动进入执行者完成实现
- **workflow.md**：唯一协调信号

---

## 提示词模板

### 启动新需求

给 **规划者（Claude）**：

```
请帮我完成以下需求：

[描述你的需求]
```

规划者会：
1. 读取 workflow.md，发现 `当前阶段 == planning`，激活规划者角色
2. 读取交接备忘了解上轮遗留上下文
3. 将需求拆成任务 YAML，写入 `task/` 目录
4. 更新 workflow.md：`当前阶段 → reviewing`
5. 在终端调用 Codex
6. **结束对话**（不等待 Codex 结果）

Codex 在终端内：
1. 自发现为审查者，审查并优化任务
2. 自动进入执行者，完成实现
3. 更新 workflow.md：`当前阶段 → planning`

---

## 典型使用场景

### 场景 1：用户发一次指令，Claude 规划 + Codex 执行完毕

1. 给规划者发需求 → 规划者写任务 + 调用 Codex → **对话结束**
2. Codex 在终端自动完成审查优化 + 执行
3. 完成后 workflow.md 回到 `planning`，用户可发新需求

### 场景 2：用户发新需求

直接给规划者发需求（workflow.md 已是 `planning` 状态）：

```
请帮我完成以下需求：

[描述你的需求]
```

## 文件说明

| 文件 | 作用 | 谁写/谁读 |
|------|------|----------|
| `workflow.md` | 协调信号（阶段 + 任务列表） | 规划者写，Codex 读写 |
| `规划者.md` | 规划者行动指南 | 规划者读 |
| `审查者.md` | 审查者行动指南 | Codex 读 |
| `执行者.md` | 执行者行动指南 | Codex 读 |
| `交接备忘.md` | 跨轮次上下文 | Codex 写，规划者读 |
| `终端调用codex.md` | Codex 调用方法 | 规划者读 |
| `task/*.yaml` | 任务定义 | 规划者写，Codex 审查+执行+回写 |
