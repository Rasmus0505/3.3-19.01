# 字段说明

- **当前阶段**：枚举值，`planning` / `reviewing` / `idle`。所有 agent 必须遵守当前阶段限制。
- **本轮待审查任务**：列表，规划者本轮产出的所有任务 ID。格式：`[TASK-xxx, TASK-yyy, ...]`。Codex 据此知道要审查哪些任务。
- **本轮用户原始背景**：字符串，规划者完整记录的用户原始需求描述（包含问题背景、目标、功能需求），供审查者理解上下文。格式：多行字符串。
- **本轮待执行任务**：列表，审查通过后待执行的任务 ID。格式：`[TASK-xxx, TASK-yyy, ...]`。执行者据此知道要认领哪些任务。
- **已完成**：列表，本轮已完成的任务及摘要。格式：`[TASK-xxx (摘要), TASK-yyy (摘要), ...]`。
- **下一步指令**：字符串，给下一个 agent 的明确指令。规划者写完后写"请在终端调用 Codex 执行审查优化和任务实现"；Codex 写"本轮完成，等待下一轮需求"或"请继续执行剩余任务"。
- **备注**：列表，跨 agent 传递的额外上下文（如用户需求）。如无，留空。
- **最后更新**：ISO 时间戳，所有 agent 写入前更新此字段。

# 自发现流程（所有 agent 必须遵守）

1. 读取本文件 `workflow.md`，获取 `当前阶段`
2. 如果 `当前阶段 == planning` → 你是**规划者**，按 `规划者.md` 行动
3. 如果 `当前阶段 == reviewing` → 你是**审查者+执行者**（Codex），按 `审查者.md` 行动，审查完成后自动进入 `执行者.md` 继续
4. 其他情况 → 无角色，等待或退出

**你不需要外部告诉你自己是哪个角色。**
**你只需要读本文件，自发现即可。**
