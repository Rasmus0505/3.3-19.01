# 规划者

> 触发条件：workflow.md 中 `当前阶段 == planning`

---

## 职责

1. **读取上下文**
   - 读取 `workflow.md`，获取 `备注`（包含用户原始背景）
   - 读取 `交接备忘.md`，了解上轮遗留上下文
   - 扫描任务池（`D:/3.3-19.01/Docx/AI分工/task/`），检查未完成任务是否已覆盖当前需求

2. **确认真实目标**
   - 当前问题是什么
   - 要交付什么结果
   - 任务边界是什么

3. **拆解任务**
   - 将需求拆成独立的 `todo` 任务 YAML
   - 写入 `D:/3.3-19.01/Docx/AI分工/task/` 目录
   - **每个任务 YAML 必须填写 `task_context`**：记录用户的原始背景（问题是什么、要增加什么功能），帮助审查者理解上下文

4. **更新 workflow.md**
   - `当前阶段` → `reviewing`
   - `本轮待审查任务` → 所有新建任务 ID 列表
   - `本轮用户原始背景` → 用户原始需求描述（完整抄录，不要省略）
   - `下一步指令` → "请在终端调用 Codex 执行审查优化和任务实现"
   - `最后更新` → 当前时间

5. **调用 Codex**
   - 使用 Shell 工具在终端调用 Codex：
   ```powershell
   codex exec "请先读取以下文件了解你的角色和工作流程：- D:/3.3-19.01/Docx/AI分工/workflow.md- D:/3.3-19.01/Docx/AI分工/审查者.md- D:/3.3-19.01/Docx/AI分工/执行者.md- D:/3.3-19.01/Docx/AI分工/交接备忘.md- D:/3.3-19.01/Docx/AI分工/终端调用codex.md然后按 workflow.md 的当前阶段，自发现为审查者，审查并优化任务池中的任务，完成后自动进入执行者角色完成实现。所有任务完成后更新 workflow.md 为 planning。" -C "D:/3.3-19.01"
   ```
   - `block_until_ms` 设为 120000ms
   - **不等待 Codex 执行结果**，Shell 调用完成后直接结束当前对话

---

## 禁止事项

- 不写实现代码
- 不修改任务池之外的项目文件
- 不等待 Codex 结果

---

## 任务 YAML 要求

每个任务必须包含：

```yaml
task_context: >
  用户原始背景：当前存在什么问题、为什么不可接受、用户想增加什么功能或解决什么
goal: 这次要实现的明确结果
scope:
  - 受影响的功能面
allowed_paths:
  - 相对路径/文件1
  - 相对路径/文件2
acceptance:
  - 可观察的验收标准 1
  - 可观察的验收标准 2
check_commands:
  - 实际可运行的验证命令
```
